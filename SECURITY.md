# Security Policy

## Scope

This repository is a public portfolio case study. It should never contain credentials, personal data, proprietary market data, protected payloads, employer-confidential information or private research artifacts.

## Reporting a concern

Please report a suspected security or privacy issue through GitHub's private vulnerability-reporting feature when available. Do not open a public issue containing a secret, token, personal information or exploit detail.

Include:

- the affected path;
- the nature of the exposure;
- reproduction steps that do not reveal additional sensitive data;
- suggested containment, when known.

## Response principles

1. Contain the exposure.
2. Revoke or rotate affected credentials outside the repository.
3. Remove sensitive material from the current tree.
4. Assess whether Git history, releases, Actions logs or forks are affected.
5. Document remediation without reproducing the sensitive content.

## Supported content

This repository contains documentation, templates and a sanitized workflow demonstration. It is not a production service and does not process live business or customer data.
